 Permissions you will be giving during authentication explained


While connecting for the first time, Cisco Spark will ask permissions to access your GitHub repositories so we can configure and manage messages received from GitHub. Per the scopes provided by GitHub, the only way we can access all your public and private repositories is using the "repo" scope which ends up giving both read and write privileges to all your public and private repo data.


Note: Although you will be giving write permission to your repo data, Cisco Spark will only use the read permission to get a list of your public and private repositories and use the write permission to configure the webhook on GitHub.

At anytime you can remove the permissions given to Cisco Spark by following the below steps.


Go to your GitHub Settings: github.com/settings.

Go to Authorized applications, click Revoke next to Cisco Spark.

Click I understand, revoke access.

 Additional Instructions


 Add a new GitHub organization to Cisco Spark anytime


Anytime you join a new organization in GitHub, you need to provide access privileges to Cisco Spark if you want to receive the notifications for that organization as well.


Follow below steps to grant access to a new organization.

Go to your GitHub Settings: github.com/settings.

Go to Authorized applications, click Cisco Spark.

Under Organization Access, click Grant Access next to your new organization.


 Permissions explained


The permissions given to Cisco Spark are read and write access to all the public and private repository data, read and write user data, full access to repository webhooks and services (no direct code access) and full access to organization webhooks (no direct access). Click here for more details.


Note: Although you have given write permission to your repo data, Cisco Spark will only use the read permission to get a list of all your public and private repositories and use the write permission to configure the webhook on GitHub.

At anytime you can remove the permissions given to Cisco Spark by following the below steps.


Go to your GitHub Settings: github.com/settings.

Go to Authorized applications, click Revoke next to Cisco Spark.

Click I understand, revoke access.
